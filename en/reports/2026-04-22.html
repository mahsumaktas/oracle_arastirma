Sections 03

🤖 KAT-2 Agent & Framework

#

Source: https://www.langchain.com/blog/running-subagents-in-the-background

  • LangChain has strongly framed the async subagent model. The start/check/update/cancel/list task model focuses on solving the supervisor deadlock problem; This could be a serious design pattern standard for agent orchestration on long missions.

Source: https://github.com/langchain-ai/agent-protocol

  • LangChain also points to Agent Protocol as the background standard. A framework-agnostic service API definition is offered on the thread, run and store axis. This reflects the need for a common language similar to “REST for agents” in agent runtimes.

Source: https://github.com/openai/openai-agents-python

  • OpenAI Agents SDK maintains open source repo momentum. The repo description keeps provider-agnostic work, sandbox agents, handoff, tracing, HITL and session management at the center.

Source: https://github.com/NousResearch/hermes-agent

  • Hermes Agent stands out as one of the powerful open source alternatives. It positions itself as "agent OS" with its emphasis on self-improving loop, skill production, multi-platform messaging gateway, cron, subagent and memory.

Source: https://github.com/lsdefine/GenericAgent, https://arxiv.org/abs/2604.17091

  • GenericAgent draws attention with its minimal core + self-evolving skill tree narrative. It puts forward the "token-efficient agent" thesis with around 3K core code, 9 atomic tools and arXiv technical report.

Sections 15

# 2) Middle post

#

The new theme in the Agent race is very clear: Runtime will win, not prompt.

In the last week:

  • OpenAI added sandbox + manifest + memory to Agents SDK
  • LangChain highlights the async subagent pattern
  • IBM published real enterprise-style agent benchmark with VAKRA

So the question is now "does the agent call the tool?" not. The issue is, does it proceed without blocking in the long run, does it carry state, does it comply with the policy?

Sources: https://openai.com/index/the-next-evolution-of-the-agents-sdk/ https://www.langchain.com/blog/running-subagents-in-the-background https://www.ibm.com/new/announcements/introducing-vakra-benchmark

# 3) Observation post

#

The 2026 agent stack is slowly becoming clear:

  • OpenAI: model-native harness + sandbox
  • LangChain: async orchestration + protocol
  • OSS side: Hermes, GenericAgent, opensre
  • Benchmark side: VAKRA

I think the next breakthrough is not "who made the best model?" It will be “who built the best working agent runtime”.
